# admin-dashboard
An intermediate HTML+CSS exercise from The Odin Project. Focuses on the use of CSS grid to create layouts

Points to note:
1. Use min-height: 100vh instead of height: 100% for body, html. With overflow: hidden, this might cause issues, although overflow is set to visible by default
2. Do not use autofit for both columns and rows. Use it only for one of them(usually columns)
3. Set linear gradient for effects in cards.
4. It's better if you do not mention any size for rows. In this case, it will only take as much space as it requires. Do not specify all row sizes with fr, this will lead to empty spaces.
5. A pixel is 1/96th of an inch. A single pixel is larger in a larger screen. This depends on the resolution too. Higher resolution more the number of pixels, more the pixels smaller will be it's size.
